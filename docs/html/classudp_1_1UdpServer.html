Detailed Description

High-rate UDP server using a pluggable ISocket.

Lifecycle:

  • Construct with a concrete ISocket and ServerConfig.
  • Call start() to spawn the worker thread and begin receiving.
  • Call stop() to request a graceful shutdown and join the thread.

Behavior:

  • The worker run_loop() uses batch I/O to pull datagrams, updates

    Stats, and (optionally) echoes payloads. Once per second it computes

    a moving per-second receive rate, exposed via last_rate_pps().

  • If ServerConfig::metrics_port is non-zero, a MetricsHttpServer

    is created and started to expose counters for scraping.

Request a graceful shutdown and join the worker thread (idempotent).

Stop the server: request loop exit and join the thread; stop metrics.

Safe to call multiple times; returns after the background thread exits.

  • Idempotent: if the thread isn’t running, calls are no-ops.
  • Ensures metrics_ is also stopped so that CI/local runs free the port.

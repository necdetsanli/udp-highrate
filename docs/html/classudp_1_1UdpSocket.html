Detailed Description

UDP socket implementation using POSIX/Linux syscalls.

Prefers batch syscalls (e.g., recvmmsg/sendmmsg) when available to reduce syscall overhead and improve packets-per-second (PPS). Falls back to classic recvfrom/sendto loops if batch syscalls are not available.

Return value counts messages, not bytes. If a datagram exceeds a buffer’s size, implementations may truncate it (consistent with recvfrom semantics).

Linux fast-path:

  • Prepares mmsghdr/iovec arrays for up to bufs.size() messages.
  • Calls recvmmsg() once; on EAGAIN/EWOULDBLOCK returns 0 (no messages).
  • On other errors returns -1; otherwise returns the number of messages received.

Fallback:

  • Performs a single recvfrom into bufs[0] and returns 1 on success,

    0 on EAGAIN/EWOULDBLOCK, or -1 on error.

Note
Return value is a message count, not bytes.

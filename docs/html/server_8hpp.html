Detailed Description

High-rate UDP server: batch receive, optional echo, runtime metrics.

This header defines the configuration and the server fa√ßade used by the

executable entry point. The server runs a background thread that:

  • receives datagrams in batches (preferring recvmmsg via ISocket),
  • updates udp::Stats counters on the hot path using relaxed atomics,
  • optionally echoes received payloads back to their sender (when enabled),
  • periodically computes a per-second receive rate (pps) for logging/inspection,
  • optionally exposes /metrics via udp::MetricsHttpServer.
Design

Core logic depends on the ISocket interface (Strategy/Port); a concrete

adapter such as UdpSocket is injected at construction time (DI). This keeps

the server testable (with MockSocket) and evolvable (future adapters).

Note
Thread-safety: one owner/thread should manage a UdpServer instance. 
  Internally the server owns a worker thread for the I/O loop. Public

  API is minimal (`start`/`stop`/getters) and not intended for concurrent

  calls without external synchronization.

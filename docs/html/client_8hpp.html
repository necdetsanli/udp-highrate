Detailed Description

High-rate UDP client: pacing + batch send with lightweight counters.

This header defines:

  • udp::ClientConfig : knobs for rate, duration, payload size, batching, etc.
  • udp::UdpClient : a small fa√ßade that sends UDP datagrams at a target 
                         packets-per-second (PPS), using batch I/O when possible.
Design

The client depends only on the ISocket interface (Strategy/Port). A concrete

adapter such as UdpSocket is injected at construction time (Dependency Injection),

which keeps the client testable (with MockSocket) and evolvable (future adapters).

Pacing & batching

The send loop distributes ClientConfig::pps over time using a monotonic clock

(see udp::now_ns). When supported by the socket implementation, messages are sent

in batches (e.g., via sendmmsg) to reduce syscall overhead and sustain high PPS.

Note
Thread-safety: a single owner should manage one UdpClient instance. 
  Internally the client owns a worker thread for the send loop. Public API

  is minimal and not intended for concurrent calls without external sync.
